# 安全策略

感谢你帮助提升 `FitnessApp` 的安全性。

## 支持范围

本项目当前以主分支的最新代码为主要维护对象。对于历史提交、长期未同步的私有部署版本或自行改造后的分支，不保证提供安全修复支持。

## 如何报告安全问题

如果你发现了可能影响用户数据、认证流程、权限控制、存储访问或部署安全的问题，请不要直接公开提交 Issue。

建议优先私下联系仓库维护者 `@wuzhan-hua`。如果你能从维护者的 GitHub 主页、仓库资料或其他公开联系入口获得邮箱或私密联系方式，请通过私下渠道发送报告。

如果你暂时找不到私下联系入口，请在公开 Issue 中仅说明“需要安全联系”或“发现潜在安全问题”，不要披露漏洞细节，等待维护者与你建立私下沟通。

报告中建议包含以下信息：

- 问题类型与影响范围
- 复现步骤
- 可能的攻击路径
- 相关截图、日志或请求样例
- 你对严重程度的初步判断

## 披露原则

在问题修复并明确允许之前，请不要：

- 在公开 Issue 中披露漏洞细节
- 发布可直接利用的 PoC
- 泄露可能影响真实用户的数据样例

## 响应说明

项目当前没有承诺固定的响应 SLA，但维护者会尽量在看到报告后尽快完成以下处理：

- 确认是否属于安全问题
- 评估影响范围与严重程度
- 给出修复或缓解计划
- 在适当时机协调公开披露

## 不属于安全问题的内容

以下情况通常不按安全漏洞处理：

- 一般性使用问题
- 功能建议
- 不影响安全边界的普通业务 Bug
- 已经需要仓库写权限或管理员权限才能利用的问题

如无法判断，也可以先按安全问题私下联系维护者，再由维护者协助分流。
